## Context

当前 NewsService 直接在 `service.py` 中硬编码了 Tushare 公告和新浪财经两个数据源。新增 6 个 Tushare API 后，需要更结构化的数据源管理方式，同时保持向后兼容。

## Goals
- 将 6 个 Tushare 新闻类 API 全部接入后端，提供统一查询接口
- 前端资讯中心支持按数据类型分 Tab 浏览，按来源筛选
- 保持现有 API 兼容性，渐进式替换旧数据源
- 各数据源独立缓存策略，避免 API 限频
- 支持部分源失败场景下的可用性降级

## Non-Goals
- 不做全文搜索引擎（仅保持现有关键词搜索）
- 不做实时 WebSocket 推送
- 不将新闻数据写入数据库（沿用现有文件缓存架构）

## Decisions

### 1. 数据源适配器模式
- **决策**：在 `service.py` 中为每个 Tushare API 创建独立的 `_fetch_xxx` 方法，统一返回 `List[NewsItem]`
- **理由**：各 API 参数和返回结构不同，独立方法便于维护；统一输出模型便于前端消费
- **替代方案**：抽象基类 + 插件体系 → 过度设计，6 个源用方法即可

### 2. 数据分类映射

| Tushare API | NewsCategory | 前端 Tab |
|---|---|---|
| `news` | FLASH | 新闻快讯 |
| `major_news` | ANALYSIS | 新闻快讯（深度标签） |
| `cctv_news` | CCTV（新增） | 新闻联播 |
| `anns_d` | ANNOUNCEMENT | 新闻快讯（公告标签） |
| `research_report` | RESEARCH（新增） | 研报数据 |
| `npr` | NPR（新增） | 政策法规 |

### 3. 缓存策略

| 数据源 | 文件缓存 TTL | 内存缓存 TTL | 拉取频率 |
|---|---|---|---|
| `news`（快讯） | 30min | 5min | 每 5 分钟 |
| `major_news`（通讯） | 2h | 30min | 每 30 分钟 |
| `cctv_news`（联播） | 24h | 2h | 每日一次 |
| `anns_d`（公告） | 2h | 30min | 每 30 分钟 |
| `research_report`（研报） | 4h | 1h | 每小时 |
| `npr`（政策） | 24h | 4h | 每日一次 |

### 4. API 限频与并发策略
- 采用**按源分组并发 + 组内串行限频**：
  - 组间并发：不同类别（快讯/联播/研报/政策）可并发
  - 组内串行：同组内多来源按顺序请求，间隔至少 1s
- 使用 `asyncio.Semaphore` 控制全局并发上限
- 失败时指数退避重试（最多 3 次）

### 5. 前端 Tab 结构调整

```
资讯中心
├── 新闻快讯（合并：news + major_news + anns_d + 可选sina兜底）
├── 新闻联播（cctv_news）
├── 机构调研（保持不变）
├── 研报数据（research_report 增强）
└── 政策法规（npr）
```

### 6. 新闻来源筛选
- 新闻快讯 Tab 内支持按来源筛选：新浪/华尔街见闻/同花顺/东方财富/财联社等
- `news` API 的 `src` 参数直接映射为筛选选项

### 7. 兜底与下线策略
- Phase 1：保留新浪数据源作为兜底，默认开启（可配置关闭）
- Phase 2：观察稳定性与覆盖率后，逐步默认关闭并最终下线

### 8. 降级与可观测性
- 单个数据源失败不影响整体响应，返回可用数据
- 响应中附带 `partial=true` 与 `failed_sources` 便于前端提示
- 记录结构化错误日志（source/api/error_code/retry_count）

### 9. 政策正文安全渲染
- `npr.content_html` 在前端渲染前必须进行 HTML 白名单净化
- 去除脚本标签、内联事件和危险 URL scheme，防止 XSS

## Risks / Trade-offs
- **API 限频风险**：通过“组间并发、组内串行+间隔”降低触发概率，但会牺牲部分实时性
- **数据量增大**：文件缓存占用增长 → 按日期自动清理 7 天前数据
- **首次加载慢**：多源请求耗时 → 前端优先展示缓存数据，后台刷新
- **源间质量不一致**：不同源字段质量不一 → 统一映射并保留原始来源字段
